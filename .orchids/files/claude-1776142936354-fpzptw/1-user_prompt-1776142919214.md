BACKEND (FastAPI — generate complete working code):

File structure:
backend/
  main.py
  routers/upload.py, jobs.py, samples.py
  pipeline/frame_extractor.py, face_detector.py, inference.py, aggregator.py, explainability.py
  models/ (ONNX model goes here)
  utils/file_handler.py, validators.py
  config.py
  requirements.txt
  Dockerfile

Endpoints to implement fully:

POST /api/upload
- Accept multipart video file
- Validate: type (mp4/mov/avi), size (≤50MB)
- Save to temp/, generate UUID job_id
- Launch BackgroundTask running full pipeline
- Return: { job_id, status: "queued" }

GET /api/job/{job_id}
- Return: { job_id, status (queued/processing/complete/failed), progress (0.0-1.0), current_step: string }
- Frontend polls this every 2 seconds

GET /api/result/{job_id}
- Return full result JSON:
  {
    job_id, verdict (REAL/DEEPFAKE/UNCERTAIN),
    confidence (0.0-1.0),
    frames_analyzed, faces_detected,
    processing_time_ms,
    suspicious_frames: [
      { frame_index, timestamp, score, image_url, artifacts: [] }
    ]
  }

GET /api/samples
- Return list of 2 sample video URLs (real + deepfake)

GET /api/health
- Return { status: ok, model_loaded: bool }

Pipeline implementation:
- frame_extractor.py: use OpenCV, sample at 2fps, max 60 frames, return list of numpy arrays + timestamps
- face_detector.py: use MediaPipe FaceDetection, return face bounding boxes per frame, skip frames with no face
- inference.py: load ONNX model, preprocess face crops to 224x224 RGB normalized, run batch inference, return per-crop scores
- aggregator.py: take top-30th percentile of scores (not mean), compute final video-level score, apply thresholds (>0.65 = DEEPFAKE, <0.35 = REAL, else UNCERTAIN), select top-5 suspicious frames
- explainability.py: rule-based artifact chip assignment — if score>0.85 assign ["face_boundary_mismatch","blending_inconsistency","texture_anomaly"], if 0.7-0.85 assign ["face_boundary_mismatch","temporal_instability"], else ["texture_anomaly"]
- Save face crop JPEGs to temp/{job_id}/ for serving
- Auto-cleanup temp files after 10 minutes
- Store job state in module-level dict: jobs: Dict[str, JobState]

NOTE: If ONNX model is not available, implement a MOCK inference function that generates plausible random scores with slight correlation to filename (so sample videos return consistent results). The mock should add realistic processing delays (sleep) at each step to make the pipeline animation meaningful.

CORS: allow all origins for hackathon
Static file serving: mount /frames route to serve temp face crop images

requirements.txt must include:
fastapi, uvicorn, python-multipart, opencv-python-headless, mediapipe, onnxruntime, numpy, Pillow, aiofiles

Dockerfile:
FROM python:3.11-slim
RUN apt-get update && apt-get install -y ffmpeg libgl1-mesa-glx libglib2.0-0
WORKDIR /app
